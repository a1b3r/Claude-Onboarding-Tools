# Getting real work done with Claude
### A hands-on orientation

Bring a real task. We're not doing toy examples.

---

## One brain, several doors

Every Claude product runs the same family of AI models.
The **interface** decides what each is good at.

---

## The tool picker

| You want to… | Reach for |
|---|---|
| Ask, learn, draft, brainstorm | **Claude Chat** |
| Hand off a messy file task | **Claude Cowork** |
| Write or fix software | **Claude Code** |
| Make slides, mockups, prototypes | **Claude Design** |

---

## The mental model

Chat = a conversation.
Cowork & Code = **leaving instructions for a capable coworker.**

You describe the outcome. It does the steps. **You review the result.**

## Prompting: the only model you need

Treat Claude like a **brilliant new colleague**:
knows a lot in general, knows *nothing* about your situation.

Brief it like you'd brief a person.

---

## The pattern (steal this)

1. **Role** — "You're the hiring manager for this job…"
2. **Process** — "…ask me questions one at a time…"
3. **Standard** — "…give honest feedback using STAR…"
4. **Guardrail** — "…don't invent anything I didn't do."

---

## Weak → strong (live demo)

**Weak:** "Write a cover letter."

**Strong:** "Write a cover letter for an entry-level marketing
analyst role at a mid-size healthcare company. I'm a recent econ
grad; resume attached. Confident but not arrogant, under 300
words, mention my internship analytics project."

---

## Exercise 1 — Prompt rewrite (20 min, pairs)

1. Each person picks a real task they brought.
2. Write the *lazy* version of the prompt. Run it.
3. Rewrite using the pattern. Run that.
4. Compare. What changed and why?

---

## The two habits that protect you

**Verify before you submit.**
AI can hallucinate — including citations that look perfectly real.

**The meeting test.**
Could you explain and defend this work without the chat open?
If not, you delegated the learning, not just the typing.
