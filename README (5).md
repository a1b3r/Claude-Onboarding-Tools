# Exercise 2 — Build a Flashcard App (solo, 25 min)

**Goal:** everyone builds working software before the break — no code knowledge
required. Demystifies "building things with AI" permanently.

## Steps
1. **(2 min)** Pick a topic you actually need to study (course material, job
   interview concepts, vocabulary — anything).
2. **(3 min)** Gather raw material: paste in notes, a syllabus section, or just
   describe the topic.
3. **(10 min)** Prompt:

```text
Create an interactive flashcard study app as an artifact from this material.
Features: show question → click to flip → "got it" / "missed it" buttons,
missed cards repeat until I get them, and a score at the end. Make it look
clean and friendly.
```

4. **(10 min)** Iterate at least twice. Ideas: shuffle mode, a timer, keyboard
   shortcuts, dark mode, harder questions. Watch how it modifies the existing
   artifact rather than starting over.

## Debrief questions
- You just specified, tested, and iterated on software. Which part felt most like
  a skill you already had?
- What would you build next? (Someone always says a budget tracker. Point them at
  the Cowork workflow instead — right tool for the job.)
