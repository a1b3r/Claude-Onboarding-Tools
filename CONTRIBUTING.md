# Contributing

Thanks for helping make this onboarding resource better. The bar for
contributions is simple: would this have helped you in your first month with
these tools?

## Ways to contribute

- **Fix or clarify docs.** Typos, confusing phrasing, broken links — small PRs
  are very welcome.
- **Add a prompt template.** Add it to the matching file in `prompts/`, follow
  the four-part pattern (role, process, standard, guardrail), and include one
  sentence on when to use it.
- **Add a Cowork workflow or Code demo.** Copy the structure of an existing
  one: a README with the brief, the expected outcome, and a review checklist.
  Include fake practice data if the exercise needs files.
- **Improve troubleshooting.** FAQs are organized by symptom ("Claude won't
  read my file"), not by feature name. Add the symptom as the heading.

## Ground rules

1. **Audience:** current undergrads of any major. No unexplained jargon.
2. **Accuracy:** don't hardcode prices, limits, or availability — link to
   official docs instead. Date-stamp anything time-sensitive.
3. **Sample data must be fake.** No real names, emails, or financial data.
4. **One change per PR** where possible, with a clear description.
5. Update `CHANGELOG.md` with your change.

## Using Claude to contribute (encouraged!)

This repo has a `CLAUDE.md` at the root, so Claude Code arrives pre-briefed on
the conventions. AI-assisted contributions are welcome — just review the output
before opening the PR. You're the contributor of record.
