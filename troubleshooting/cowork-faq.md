# FAQ — Claude Cowork

> 📖 Pairs with [Module 04](../guide/index.html#m4).

## "I can't find Cowork / don't have access"
Cowork lives in the Claude **desktop app** (not the website) and availability varies
by plan and platform — it launched as a research preview and access has been
expanding. Check the current state: https://claude.com/product/cowork and
https://support.claude.com.

## "It can't see my files"
Cowork only sees folders you've explicitly granted. Confirm you granted the right
folder (a parent folder doesn't automatically include cloud-synced placeholders —
files that live "online-only" in Dropbox/OneDrive/iCloud may need to be downloaded
locally first).

## "The task stalled or is taking forever"
Big folders and vague briefs both cause wandering. Stop the task and restart with:
a tighter scope (subfolder), a clearer outcome, and explicit rules for ambiguity
("if unsure, flag it and move on rather than investigating"). Long multi-step tasks
also consume usage limits — see your plan's details if tasks consistently can't finish.

## "It changed/moved something I didn't want — can I undo?"
First: stop the task. Then check whether your OS safety nets have it (Trash/Recycle
Bin, Time Machine/File History, cloud version history). Then ask Cowork itself to
list everything it changed and reverse specific moves — it knows what it did within
the session. **Prevention is the real answer:** backups before big jobs, "don't
delete — quarantine" rules in every brief, and a small test run first
(see [`../cowork-workflows/`](../cowork-workflows/)).

## "The output has errors"
Treat Cowork like a fast junior colleague: the work arrives done, but you review it.
Numbers especially — spot-check against sources every time. If errors repeat, the
fix is usually in the brief: add explicit rules ("never guess amounts; flag
unreadable values").
