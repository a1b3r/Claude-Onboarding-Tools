# Workflow 01 — Downloads Folder Cleanup

**Situation:** a folder of accumulated files with chaotic names and no structure.
**Practice data:** `sample-files/01-downloads-cleanup/` (copy it out of the repo first).

## The brief (adapt the bracketed parts)

```text
Organize this folder. Create subfolders by category that make sense for what's
actually here (e.g., finances, job search, school, misc), move each file into
the right one, and rename files to a consistent, readable pattern:
YYYY-MM-DD-description.ext where a date is known, otherwise just description.ext.

Rules:
- Don't delete anything. If something seems like junk, move it to a folder
  called _review-before-deleting instead.
- If you're unsure where a file belongs, put it in misc and list it in your
  summary rather than guessing.
- When done, give me a summary: the folder tree, every rename (old → new),
  and anything you flagged.
```

## Expected outcome

A tidy folder tree, consistently named files, a `_review-before-deleting` folder
instead of deletions, and a summary you can audit.

## Review checklist

- [ ] Nothing was deleted (count files before and after — should match).
- [ ] Spot-check 3 renames: does the new name actually describe the file?
- [ ] Open `_review-before-deleting` — do you agree with what landed there?
- [ ] Could you find a specific file faster now than before? (The whole point.)

**The transferable rule:** "don't delete — quarantine" is the single best line to put
in any file-organizing brief.
