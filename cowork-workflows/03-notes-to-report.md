# Workflow 03 — Meeting Notes → Status Report

**Situation:** weeks of scrappy meeting notes need to become a one-page status report
for someone who attended none of the meetings.
**Practice data:** `sample-files/03-meeting-notes/` (copy it out of the repo first).

## The brief

```text
Read all the meeting notes in this folder (a student events committee planning
a spring fundraiser). Write status-report.md, one page, for a faculty advisor
who hasn't attended any meetings:

1. Where the project stands overall (2-3 sentences)
2. Decisions made (with the date each was decided)
3. Open questions / risks
4. Action items: owner, task, due date — in a table

Rules:
- Only include things actually stated in the notes. If notes conflict, use
  the most recent and footnote the conflict.
- Don't invent owners or dates that aren't written down — mark them "TBD".
- Keep the original notes unmodified.
```

## Expected outcome

A report a stranger could read in 3 minutes, with every claim traceable to the notes.

## Review checklist

- [ ] Pick 2 "decisions" in the report and find them in the source notes.
- [ ] The planted trap: two notes files contradict each other about the venue. The
      report should surface this (most recent wins, conflict footnoted) — not silently
      pick one.
- [ ] Action items with no stated owner are marked TBD, not assigned by guesswork.

**The transferable rule:** "only what's actually in the sources, conflicts surfaced"
turns Cowork from a fluent summarizer into a trustworthy one.
